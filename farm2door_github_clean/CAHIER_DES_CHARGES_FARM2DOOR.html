Cahier des charges - Farm2Door

Projet : Farm2Door

Type : Projet de Fin d'Annee

Domaine : Marketplace agricole locale

Technologies : Django, Python, SQLite, HTML, CSS

Document sans diagrammes.

1. Introduction

Farm2Door est une application web qui met en relation les agriculteurs locaux et les clients. L'objectif principal est de faciliter la vente directe des produits agricoles frais, avec une experience simple pour le client et un espace de gestion pratique pour l'agriculteur.

2. Contexte du projet

Les petits producteurs agricoles rencontrent souvent des difficultes pour vendre leurs produits directement aux consommateurs. Les clients cherchent une solution simple pour acceder a des produits frais, connaitre les prix, verifier la disponibilite et passer commande rapidement.

3. Problematique

Comment creer une application web permettant aux agriculteurs de publier leurs produits, aux clients de commander facilement, a l'administrateur de suivre l'activite et au systeme de generer une preuve d'achat claire apres chaque commande ?

4. Objectifs

  • Creer une page d'accueil claire et professionnelle.
  • Afficher un catalogue de produits agricoles.
  • Permettre l'inscription et la connexion des utilisateurs.
  • Gerer les roles client et agriculteur.
  • Permettre aux agriculteurs d'ajouter, modifier et supprimer des produits.
  • Permettre aux clients de gerer un panier.
  • Ajouter une etape de paiement simule.
  • Generer un bon d'achat PDF apres finalisation.
  • Fournir des dashboards client, farmer et admin.

5. Acteurs

Client

Consulte les produits, ajoute au panier, choisit le paiement, finalise une commande, consulte son historique et telecharge ses bons d'achat.

Agriculteur

Ajoute ses produits, gere les images, les prix et les stocks, consulte les commandes recues et suit ses revenus.

Administrateur

Suit les utilisateurs, produits, commandes, revenus et peut gerer les donnees depuis Django Admin.

6. Besoins fonctionnels

  • Authentification et gestion des roles.
  • Gestion des produits et images.
  • Catalogue produits.
  • Panier avec quantites et total.
  • Paiement a la livraison ou par carte simulee.
  • Creation de commandes.
  • Mise a jour automatique du stock.
  • Generation de bon d'achat PDF.
  • Historique client.
  • Commandes recues farmer.
  • Dashboard administrateur.

7. Besoins non fonctionnels

  • Interface simple et ergonomique.
  • Pages protegees selon le role de l'utilisateur.
  • Application rapide en environnement local.
  • Code organise selon la structure Django.
  • Projet facile a presenter et a maintenir.

8. Technologies utilisees

  • Backend : Python, Django.
  • Base de donnees : SQLite.
  • Frontend : HTML, CSS, templates Django.
  • Administration : Django Admin.
  • Media : images produits stockees dans le dossier media.

10. Regles de gestion

  • Un produit appartient a un agriculteur.
  • Un client peut passer plusieurs commandes.
  • Une commande contient plusieurs produits.
  • Le stock diminue apres validation.
  • Le paiement par carte marque la commande comme payee.
  • Le paiement a la livraison marque la commande comme en attente.
  • Le bon PDF est genere apres finalisation.
  • Un agriculteur gere uniquement ses propres produits.

11. Tests a effectuer

  • Creer un compte client et farmer.
  • Ajouter un produit avec image.
  • Modifier et supprimer un produit.
  • Ajouter des produits au panier.
  • Finaliser une commande.
  • Tester le paiement a la livraison.
  • Tester le paiement par carte simulee.
  • Telecharger le bon d'achat PDF.
  • Verifier les dashboards.

12. Conclusion

Farm2Door est une application web complete pour un PFA de marketplace agricole locale. Elle couvre le parcours client, le parcours agriculteur et le suivi administrateur. Le projet peut evoluer vers le paiement reel, les notifications, les categories, les avis clients et le suivi de livraison.
